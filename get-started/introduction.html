Activity, Service, Receiver

Android has three main file types: Activity, Service, and receivers.
Activity: is UI based, "it is the java for the window". "An activity represents a single screen with a user interface just like window."
Service: is a background service, it has no ui and can be ran/running after the user closes UI compenent of the application.
Receiver: are listeners, they allow you to "catch" incoming actions. Ex. Notifications (You dont pull from the server every second, it's a push to the phone and the receiver catches it).

Android Lifecycle

Open the more complex lifecycle with Fragments.

Summary of some of the lifecycle states in my words.

OnCreate: This method is trigger when you open a window for the first time. The building of the UI Windows, and listeners are in this method.
OnResume: This method is trigger when you open a window for the first time, and when you return to app that was in the background (After you maximize the app).
OnDestroy: This method is trigger when you close the app completely.

Android Manifest

Every application must have an AndroidManifest.xml file (with precisely that name) in its root directory. The manifest file provides essential information about your app to the Android system, which the system must have before it can run any of the app's code.[Source]

The Android Manifest contains information about the activities (windows), services, receivers, permissions, app name, app package name, etc. in your app. This is the roadmap for the application.
